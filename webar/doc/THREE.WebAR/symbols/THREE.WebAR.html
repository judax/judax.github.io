Namespace Detail
THREE.WebAR
The WebAR namespace inside the THREE namespace. Inside this namespace, different utilities to be able to handle WebAR functionalities on top of the ThreeJS framework/engine. NOTE: As a coding standard all the variables/functions starting with an underscore '_' are considered as private and should not be used/called outside of the namespace/class they are defined in.

<static> {THREE.Camera} THREE.WebAR.createVRSeeThroughCamera(vrDisplay, near, far)
A utility function to create a THREE.Camera instance with as frustum that is obtainer from the underlying vrdisplay see through camera information. This camera can be used to correctly render 3D objects on top of the underlying camera image.
Parameters:
{VRDisplay} vrDisplay
- The VRDisplay that is capable to provide a correct VRSeeThroughCamera instance in order to obtain the camera lens information and create the correct projection matrix/frustum.
{number} near
- The near plane value to be used to create the correct projection matrix frustum.
{number} far
- The far plane value to be used to create the correct projection matrix frustum.
Returns:
{THREE.Camera} - A camera instance to be used to correctly render a scene on top of the camera video feed.
<static> {THREE.Mesh} THREE.WebAR.createVRSeeThroughCameraMesh(vrDisplay)
A utility function that helps create a THREE.Mesh instance to be able to show the VRSeeThroughCamera as a background quad with the correct texture coordinates and a THREE.VideoTexture instance.
Parameters:
{VRDisplay} vrDisplay
- The VRDisplay that is capable to provide a correct VRSeeThroughCamera instance.
Returns:
{THREE.Mesh} - The THREE.Mesh instance that represents a quad to be able to present the see through camera.
<static> {number} THREE.WebAR.getIndexFromOrientation(orientation)
Provides an index based on an orientation angle. The corresponding index to the angle is: orientation = 0 <-> index = 0 orientation = 90 <-> index = 1 orientation = 180 <-> index = 2 orientation = 270 <-> index = 3
Parameters:
{number} orientation
- The orientation angle. Values are: 0, 90, 180, 270.
Returns:
{number} - An index from 0 to 3 that corresponds to the give orientation angle.
<static> {number} THREE.WebAR.getIndexFromScreenAndSeeThroughCameraOrientations(vrDisplay)
Returns an index that is based on the combination between the display orientation and the see through camera orientation. This index will always be device natural orientation independent.
Parameters:
{VRDisplay} vrDisplay
- The VRDisplay that is capable to provide a correct VRSeeThroughCamera instance.
Returns:
{number} - The index from 0 to 3 that represents the combination of the device and see through camera orientations.
<static> THREE.WebAR.positionAndRotateObject3DWithPickingPointAndPlaneInPointCloud(pointAndPlane, object3d, scale)
Transform a given THREE.Object3D instance to be correctly positioned and oriented according to a given VRPickingPointAndPlane and a scale (half the size of the object3d).
Parameters:
{VRPickingPointandPlane} pointAndPlane
- The point and plane retrieved using the VRDisplay.getPickingPointAndPlaneInPointCloud function.
{THREE.Object3D} object3d
- The object3d to be transformed so it is positioned and oriented according to the given point and plane.
{number} scale
- The value the object3d will be positioned in the direction of the normal of the plane to be correctly positioned. Objects usually have their position value referenced as the center of the geometry. In this case, positioning the object in the picking point would lead to have the object3d positioned in the plane, not on top of it. this scale value will allow to correctly position the object in the picking point and in the direction of the normal of the plane. Half the size of the object3d would be a correct value in this case.

<static> THREE.WebAR.updateVector3Orientation(vrDisplay, v)
Updates a vector 3D by rotating it depending on the orientation of the current screen and the see through camera. This method can be used to correctly rotate the touch X, Y position when picking. This method assumes that the vector only holds a 2D position (X, Y) that is also normalized (values from 0 to 1) as it rotates around the center (0.5).
Parameters:
{VRDisplay} vrDisplay
- The VRDisplay that holds the VRSeeThroughCamera.
{THREE.Vector3} v
- The ThreeJS vector3 that holds a normalized 2D position, supposedly the X, Y normalized position of a touch point for picking purposes.
